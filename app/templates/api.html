BGP Looking Glass API

Welcome to the BGP Looking Glass API documentation. This API allows you to execute network commands on configured devices and retrieve the raw output in JSON format. Below are some example curl commands to interact with the API endpoint at /api/execute.

API Endpoint

All API requests should be sent to /api/execute using the POST method with a JSON payload.

Example Curl Commands

1. Execute a Simple Command (No Variables)

curl -X POST -H "Content-Type: application/json" -d '{"device": "router1", "command": "show_bgp_summary"}' http://your-server-address/api/execute

2. Execute a Command with a Single Variable (Ping)

curl -X POST -H "Content-Type: application/json" -d '{"device": "router2", "command": "ping", "variables": {"host": "8.8.8.8"}}' http://your-server-address/api/execute

3. Execute a Command with a BGP Neighbor IP

curl -X POST -H "Content-Type: application/json" -d '{"device": "router1", "command": "show_bgp_neighbor", "variables": {"neighbor": "192.168.1.1"}}' http://your-server-address/api/execute

4. Execute a Command with an IP Prefix

curl -X POST -H "Content-Type: application/json" -d '{"device": "router2", "command": "show_ip_bgp_prefix", "variables": {"prefix": "10.0.0.0/24"}}' http://your-server-address/api/execute

5. Execute a Command with a Community Value

curl -X POST -H "Content-Type: application/json" -d '{"device": "router1", "command": "show_ip_bgp_community", "variables": {"community": "11:11"}}' http://your-server-address/api/execute

Response Format

The API will return a JSON response. If successful, the response will include the raw output from the device. If there is an error, it will return an error message.

Success Response:
{
    "result": "Raw output from the device command..."
}

Error Response:
{
    "error": "Error message describing the issue"
}

Notes

  • Replace your-server-address with the actual address of your server (e.g., localhost:5000 or your domain).
  • Ensure the device and command are valid and allowed for the specified device.
  • Variables must match the expected format for the command as defined in the configuration.
